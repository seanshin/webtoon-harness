# 🎬 Webtoon Studio — UX 기획안

> 로컬 웹 GUI로 **원작(소설/문장) 재구성 → 생성(렌더) → 조립**까지 한 흐름에서 구동하는 웹툰 제작 스튜디오.
> 기존 하네스 자산(`weru_imagegen.py`, 비트시트 생성기, `build_viewer.py`)을 FastAPI 백엔드로 감싸 재사용하고,
> **WeRU.B 서버의 LLM 기능군(`/api/novel`·`/api/story`·`/api/korean`·`/api/vision`)을 적극 활용**한다.
> 운영 모델 = **하이브리드**: 기본은 자동 진행, 중요한 두 지점(재구성 확정 · 최종 조립)에서만 사용자 확인.
>
> **핵심 컨셉**: "백지에서 쓰는" 도구가 아니라, **이미 있는 소설·시놉시스·문장 뭉치를 웹툰 컷으로 재구성**하는 각색(adaptation) 도구. 집필은 보조, 재구성이 본류.

---

## 1. 목표와 설계 원칙

- **한 흐름, 끊김 없음**: 빈 화면 → 완성된 세로 스크롤 뷰어까지 탭 이동 없이 좌측 파이프라인 레일로 이동.
- **사람은 결정만, 기계는 반복을**: 창작 결정(스토리·캐릭터 룩·최종 검수)은 사람이, 70컷 프롬프트 합성·렌더·조립은 자동.
- **긴 렌더를 견디는 UX**: 35분 렌더 동안 실시간 썸네일·진행률·ETA·부분 재렌더. "제출하고 방치"가 아니라 "지켜보며 손봄".
- **항상 편집 가능**: 자동 생성물(비트·프롬프트·말풍선)은 전부 인라인 편집 가능. 자동은 초안일 뿐.
- **자가완결 산출물**: 최종은 base64 임베드 단일 HTML — 어디서 열어도 안 깨짐(검증된 패턴).

## 2. 정보구조 — 6단계 파이프라인 + 하이브리드 게이트

```
 ┌─ 좌측 파이프라인 레일 (항상 표시, 현재 위치/게이트 상태) ──────────────────┐
 │                                                                            │
 │ ①원작·재구성 → ②스토리보드 → ③캐스트 → ④렌더 → ⑤레터링·조립 → ⑥익스포트 │
 │    [GATE]         auto       auto/GATE?  auto        [GATE]                 │
 └────────────────────────────────────────────────────────────────────────────┘

 ● GATE = 사용자 확인 후 진행 (하이브리드 정지점)
   - ① 재구성 확정 게이트  : 원작 인제스트→자동 분석→웹툰 컷 재구성. 컷/대사가 만족스러울 때까지 반복 → "확정"
   - ⑤ 최종 조립 게이트    : 말풍선 위치/뷰어 확인 후 "조립·익스포트"
 ● 그 외(②③④)는 자동 진행하되 각 화면에서 언제든 수정 가능
   - ③ 캐스트는 원작 분석으로 **자동 채움**(인물 추출). 신규/모호 인물만 "룩 확정" 옵션 게이트
```

자동 흐름의 의미: ①에서 원작을 넣고 "확정"을 누르면 ②비트→샷리스트→프롬프트 합성, ③(원작에서 추출된)인물 레퍼런스 등록, ④렌더가 **연쇄로 자동 실행**되고, 사용자는 ④렌더 콘솔에서 결과를 지켜보다 ⑤에서 멈춰 최종 검수한다.

## 3. 화면별 설계

### ① 원작·재구성 스튜디오 ★ (소설/문장 → 웹툰 컷)

**왼쪽=원작 소스 텍스트, 오른쪽=재구성된 웹툰 비트.** 가운데에 WeRU.B 분석 결과를 띄워, 원문을 컷으로 "각색"하는 화면. 백지 집필이 아니라 **있는 글을 재구성**하는 게 본류.

```
┌ Webtoon Studio ─ 심야 편의점 / 2화 ──────────────────────[ 저장됨 ]─┐
│레일│ ┌─ 원작 소스 ───────┐ ┌─ 분석(WeRU.B) ──┐ ┌─ 웹툰 비트(재구성)──┐│
│ ①◉│ │ [📄소설 업로드]    │ │ 👥 인물 자동추출  │ │ #1 [wide][EXT] 🅝   ││
│ ② │ │ [📋붙여넣기]       │ │  지호·유나·…→③   │ │  비 오는 새벽 3시…  ││
│ ③ │ │ ───────────────  │ │ 🎭 감정맵 ▁▂▆█▅ │ │ ─────────────────  ││
│ ④ │ │ "3시였다. 비가     │ │ ⏱ 페이싱: 23화중 │ │ #2 [full]🧍지호 💭  ││
│ ⑤ │ │  내렸고, 늘 그 여자 │ │  이 장면=2.3컷/문 │ │  오늘도… 조용하네.  ││
│ ⑥ │ │  가 들어왔다…"     │ │ 🧶 떡밥 3건 감지  │ │ ─────────────────  ││
│   │ │ (원문 스크롤)      │ │ 🗣 말투: 지호=건조 │ │ #3 [emotion]🧍 😱   ││
│   │ │                   │ │     유나=느릿·존대 │ │  어떻게…            ││
│   │ │ ▼ 챕터 선택        │ │ 🌍 장소: 편의점/   │ │ [+대사][샷▼][🗑]    ││
│   │ │  [3화: 단골 ▼]     │ │     골목 → LOC_*  │ │ (원문 23문장→68컷)  ││
│   │ └───────────────────┘ └──────────────────┘ └────────────────────┘│
│   │ ┌─ 재구성 액션 (선택 구간/전체) ──────────────────────────────────┐│
│   │ │ [✦ 컷으로 재구성] [↔분량 압축:68컷] [말투 입히기] [톤:차갑게]    ││
│   │ │ 부분: • 이 문단 더 쪼개 • 침묵 컷 추가 • 이 대사 유나 말투로      ││
│   │ └────────────────────────────────────────────────────────────────┘│
│   │                       [ 임시저장 ]   [ 재구성 확정 → ② ] 🔒GATE    │
└──────────────────────────────────────────────────────────────────────┘
```

**재구성 흐름 (WeRU.B 기능 매핑):**
1. **인제스트** — 소설 파일 업로드 / 문장 붙여넣기 → `POST /api/novel/decompose/upload` (서버가 `project_id` 발급, 챕터/장면 분해) · `/api/novel/extract/metadata`.
2. **자동 분석(가운데 패널)** — 확정 전에 원작을 읽어낸다:
   - `POST /api/novel/analyze/character` → **인물 자동 추출 → ③캐스트 자동 채움**.
   - `/api/novel/analyze/emotion_map` → 감정 곡선(강조 컷·침묵 컷 배치 근거).
   - `/api/novel/analyze/pacing` → 문장당 컷 밀도(분량 환산: 23문장 챕터 → 68컷).
   - `/api/korean/character-voice` `{text, characters[]}` → **캐릭터별 한국어 말투** → 대사 일관성.
   - `/api/story/foreshadowing/detect` → **떡밥 감지**(회차 넘는 복선 추적).
   - `/api/novel/analyze/worldbuilding` → 장소 추출 → `LOC_*` 토큰 자동 생성.
3. **컷으로 재구성** — 선택 구간 또는 챕터를:
   - `/api/novel/chapter/condense` `{project_id, chapter_num, target_words}` → 긴 원작을 **웹툰 분량으로 압축**.
   - `/api/novel/outline/generate` + `/api/novel/scene/dialogue`·`/scene/description`·`/scene/transition` → 장면을 **컷 단위 비트**(shot/loc/char/대사)로 분해.
   - `/api/korean/style-transfer` → 원작 문어체 대사를 **웹툰 말풍선 톤**으로 변환.
4. **비트 카드 편집** — 오른쪽 컷은 전부 인라인 편집(shot/장소/대사 type·speaker·한글). "이 문단 더 쪼개", "이 대사 유나 말투로" 같은 부분 재구성.
5. **게이트** — "재구성 확정"이 ②~④ 자동 연쇄의 방아쇠.

> 백지 집필이 필요할 땐 보조로 `/api/story/generate`·`/api/novel/chapter/write`를 쓰되, 화면의 1차 동작은 항상 "원작→재구성".

### ② 스토리보드 / 샷리스트 (자동, 편집 가능)

비트 → 70컷 그리드. 컷마다 자동 합성된 **프롬프트**와 **IP-Adapter 배정**을 보여주고 손볼 수 있다.

```
┌ 스토리보드 (70컷) ───────────────── [모델 일괄변경] [재합성] ─┐
│ #1 wide  EXT      | 프롬프트 ▸ "...neon storefront..."  [no-id]│
│ #2 full  CVS 지호 | 프롬프트 ▸ "...gray hoodie..."  [id:지호/full0.4]│
│ #3 emo   CVS 지호 | 프롬프트 ▸ "...terror..."     [id:지호/face0.5]│
│ #15 ins  CVS      | 프롬프트 ▸ "...still-life...단일객체" [no-id] ⚠소품│
│   ...                                                          │
│ • 샷 바꾸면 IP-Adapter 강도 자동 갱신(wide/full→0.4, close→0.5) │
│ • '말풍선 자리' 힌트로 여백 위치 자동 지정(긍정 프롬프트 bubble 금지)│
│ • ⚠ 소품 인서트는 자동으로 "객체강제" 프롬프트 프리셋 적용        │
└────────────────────────────────────────────────────────────────┘
```

### ③ 캐스트 (레퍼런스/character_id) — 원작에서 자동 채움

①의 `novel/analyze/character` 결과로 **인물·외형·말투가 자동 입력**된다. 사용자는 레퍼런스 룩만 확정.

```
┌ 캐스트 (원작에서 추출 ✓) ───────────────────────────┐
│ 🧍 지호  id:e95ac39e…  말투:건조  [정면레퍼런스 ✓] [재렌더]│
│ 👻 유나  id:08f8bba9…  말투:느릿  [정면레퍼런스 ✓] [재렌더]│
│ ◐ 사장   (추출됨, 미등록)  외형토큰[자동제안] [레퍼런스 렌더→등록]│
│   ↳ 외형 모호/신규만 '룩 확정' 옵션 게이트                │
└──────────────────────────────────────────────────────┘
```

- 외형 토큰은 원작 묘사에서 자동 제안(`analyze/character`), 부족하면 사용자가 보강 후 `character/create` 등록.
- 말투(`korean/character-voice`)는 ⑤레터링에서 그 인물 대사에 자동 반영.

### ④ 렌더 콘솔 ★ (생성 — 긴 렌더를 지켜보는 화면)

70셀 그리드가 렌더되는 대로 실시간 썸네일로 채워진다. WebSocket으로 상태 스트리밍.

```
┌ 렌더 ─ animagine HQ ──────  ▓▓▓▓▓▓▓░░░  43/70  ETA 13:20 ─┐
│ ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐   상태 범례               │
│ │🖼││🖼││🖼││⏳││ ││ ││✕│   🖼완료 ⏳렌더중 □대기 ✕실패│
│ └──┘└──┘└──┘└──┘└──┘└──┘└──┘                              │
│ ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐   • 완료 셀 클릭 → 원본 확대 │
│ │🖼││🖼││🖼││🖼││🖼││🖼││🖼│   • 실패 셀 → [재렌더]      │
│ └──┘└──┘└──┘└──┘└──┘└──┘└──┘   • 마음에 안 듦 → [프롬프트 │
│   ...                            손보고 재렌더]            │
│ [⏸ 일시정지] [실패 5개 일괄 재렌더] [소품컷 객체강제 재렌더] │
└────────────────────────────────────────────────────────────┘
```

- **실시간 채움**: 헬퍼는 폴링 후 일괄 다운로드지만, 콘솔은 `/status`를 구독해 **완료되는 즉시 썸네일** 표시(기존 `.jobids.json` 직접조회 패턴을 백엔드가 수행).
- **부분 재렌더**: 실패/불만 셀만 프롬프트·시드 손봐 재제출. 전체를 다시 돌리지 않는다.
- **품질 토글**: draft로 빠르게 전체 → 확정 후 hq로 최종만.

### ⑤ 레터링 & 조립 ★ (드래그 말풍선 — 최종 게이트)

패널 위에 말풍선을 **드래그로 배치**하고 한글을 인라인 수정. 오버레이는 100% 정확·수정 자유(이미지에 안 굽힘).

```
┌ 레터링 ─ #2 ────────────────[ 미리보기 ] [ 다음 ▶ ]─┐
│  ┌─────────────┐   ┌─ 말풍선 속성 ──────────┐       │
│  │  [패널 작화]  │   │ type [thought ▼]       │       │
│  │   ╭──────╮   │   │ speaker [지호]         │       │
│  │   │오늘도…│◀──┼── │ 한글 [오늘도… 조용하네.]│      │
│  │   ╰──────╯   │   │ 꼬리 [bottom-left ▼]   │       │
│  │      🧍      │   │ 폭 [52%]  ⟵드래그로 이동│       │
│  └─────────────┘   └────────────────────────┘       │
│  • 얼굴/핵심작화 가리면 경고 표시 → 위치 미세조정     │
│              [ 최종 조립 · 익스포트 → ⑥ ] 🔒GATE     │
└──────────────────────────────────────────────────────┘
```

- **비전 보조 자동배치**: 렌더 직후 백엔드가 각 패널을 `/api/vision/describe`(+`/api/detect/image`)로 읽어 **인물 얼굴·핵심 피사체 위치를 추정** → 말풍선 초기 위치를 그 영역을 피해 자동 배치한다. 사용자는 미세조정만(기존 수동 `bubble-space`를 비전이 보조).
- **말풍선 텍스트**: ①에서 `korean/character-voice`로 잡은 말투가 이미 입혀진 한글이 들어오며 인라인 수정 자유(오버레이라 100% 정확).

│  /api/lettering         lettering.json 편집 (+얼굴회피 힌트)│
│  /api/assemble          → build_viewer.py 래핑             │
│  재사용: gen_jobs(비트→jobs), build_viewer, weru_imagegen  │
└───────┬───────────────────────────────────────────────────┘
        │ SSH 단접속(submit/status/download) + HTTP(127.0.0.1:8585)
┌───────▼───────────────────────────────────────────────────┐
│ WeRU.B GPU 서버 (단일 백엔드 — 539 endpoints)              │
│  · 이미지: image/generate · character/create · IP-Adapter  │
│  · 소설:  novel/decompose · analyze · condense · scene     │
│  · 스토리: story/foreshadowing · world · continuity        │
│  · 한국어: korean/character-voice · style-transfer         │
│  · 비전:  vision/describe · detect/image                   │
│  · 번역:  translate · writing/tones                        │
└───────────────────────────────────────────────────────────┘
```

- **백엔드는 얇은 오케스트레이터**: 텍스트(재구성)도 이미지(렌더)도 **WeRU.B 단일 서버**가 처리한다. 외부 LLM(Claude API) 없이 자급 — 로컬·오프라인·무과금. 백엔드는 검증된 `weru_imagegen.py`·`build_viewer.py`를 모듈로 호출하고 novel/story/korean/vision 호출을 조율할 뿐.
- **텍스트 엔진 = WeRU.B `/api/novel`·`/api/story`·`/api/korean`** (집필/분석/재구성). 외부 API 의존 0.
- **이미지 엔진 = WeRU.B `/api/image`** (검증된 IP-Adapter 일관성).
- **비전 QA 루프 = WeRU.B `/api/vision/describe`** — 렌더 결과를 다시 서버가 읽어 자가 검증(아래 §활용 참조).
- **상태 저장**: 회차당 `project.json` + 서버측 novel `project_id` 연결. 단일 진실원천.

## 6-1. WeRU.B 서버 기능 적극 활용 (텍스트·비전)

스튜디오는 이미지 외에 서버의 LLM 기능군을 1급으로 활용한다. 외부 API 0, 로컬 자급.

| 스튜디오 기능 | WeRU.B 엔드포인트 | 효과 |
|---------------|-------------------|------|
| 원작 인제스트 | `POST /api/novel/decompose/upload`·`/extract/metadata` | 소설/문장 → `project_id`, 챕터·장면 분해 |
| 인물 자동 추출 → ③캐스트 | `POST /api/novel/analyze/character` | 등장인물·외형·관계를 캐스트로 자동 채움 |
| 감정 곡선 | `/api/novel/analyze/emotion_map` | 강조/침묵 컷 배치 근거 |
| 페이싱·분량 환산 | `/api/novel/analyze/pacing` | 문장↔컷 밀도 → 70컷 분배 |
| 장소 토큰(LOC_*) | `/api/novel/analyze/worldbuilding` | 배경 연속성 토큰 자동 |
| 캐릭터 말투 | `POST /api/korean/character-voice` `{text,characters[]}` | 인물별 한국어 말풍선 톤 일관 |
| 대사 문체 변환 | `/api/korean/style-transfer` | 원작 문어체 → 웹툰 구어체 |
| 분량 압축 | `POST /api/novel/chapter/condense` `{project_id,chapter_num,target_words}` | 긴 원작 → 웹툰 분량 |
| 장면→컷 분해 | `/api/novel/outline/generate`·`/api/novel/scene/{dialogue,description,transition}` | 비트 자동 생성 |
| 떡밥/복선 | `POST /api/story/foreshadowing/detect`·`/add`·`/resolve` | 회차 넘는 복선 추적 |
| 연속성(다회차) | `/api/story/world/*`·`/api/story/chain/{id}/summarize`·`continuity` | 세계관/관계도 승계 |
| 패널 자가검증 | `POST /api/vision/describe` (이미지 업로드) | 렌더가 비트 의도와 맞는지 서버가 재확인 |
| 말풍선 자동배치 | `/api/detect/image`·`/api/vision/describe` | 얼굴/피사체 회피 위치 추정 |
| 번역(다국어 출간) | `/api/translate`·`/api/writing/translate` | 오버레이만 교체해 다국어판 |

**비전 QA 루프(차별화 포인트):** ④렌더 → 백엔드가 각 패널을 `/api/vision/describe`로 읽어 비트 설명과 대조 → 불일치(예: "신문 인서트인데 인물 렌더됨") 자동 플래그 → ④콘솔에 "⚠ 의도불일치" 배지 + 객체강제 재렌더 제안. 사람이 70컷을 눈으로 다 안 봐도 된다.

## 7. 구축 로드맵 (MVP → 풀)

- **M1 — 골격(읽기전용 파이프라인)**: project.json 로드 → ④렌더 콘솔(WebSocket 진행) + ⑥미리보기. 기존 산출물을 웹으로 "보는" 단계.
- **M2 — 생성 루프**: ③캐스트 등록 + ④부분 재렌더 + ⑤말풍선 드래그 편집 + 조립 + **비전 QA 루프**(`vision/describe`). "손보고 다시 굽는" 핵심 UX.
- **M3 — 원작 재구성**: ①원작 인제스트(`novel/decompose`) + 자동 분석(`analyze/*`·`korean/*`·`foreshadowing`) + 재구성(`condense`·`scene/*`) + 하이브리드 게이트 연쇄. "소설부터" 완성.
- **M4 — 다회차/운영**: 대시보드, 연속성 승계(`story/world`·`chain`), 다국어(`translate`), 배포·프리셋.

> 권장: **M1→M2 먼저**(이미 검증된 렌더/조립을 GUI로 감싸 즉시 효용) → M3 재구성 추가. 텍스트·이미지 모두 WeRU.B라 신규 엔드포인트는 호출만 하면 됨.

## 8. 결정 필요/리스크

- **프론트 스택**: htmx+Alpine(가벼움) vs React/Vue(드래그·그리드 풍부). 드래그 말풍선·실시간 그리드 비중 크면 Vue 고려.
- **텍스트 엔진**: WeRU.B `/api/novel`·`/api/story`·`/api/korean`로 자급(외부 API 0) — 확정. 모델 품질이 부족한 특정 작업만 선택적 보강 검토.
- **WeRU.B 동시성**: 서버는 순차 처리(1 GPU). 텍스트+이미지+비전이 한 서버 큐를 공유 → 우선순위/큐 정책 필요(렌더 중 분석 호출 지연 가능).
- **원작 저작권**: 외부 소설 인제스트 시 2차 창작/배포 권리 확인은 사용자 책임. 스튜디오는 출처(`source`)를 회차 메타에 기록.
- **인증/배포**: 로컬 단독 vs 사내 공유(공유면 간단 인증·프로젝트 격리).
```
