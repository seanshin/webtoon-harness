# 🎬 Webtoon Studio — UX 기획안

> 로컬 웹 GUI로 **스크립트 작성 → 생성(렌더) → 조립**까지 한 흐름에서 구동하는 웹툰 제작 스튜디오.
> 기존 하네스 자산(`weru_imagegen.py`, 비트시트 생성기, `build_viewer.py`)을 FastAPI 백엔드로 감싸 재사용한다.
> 운영 모델 = **하이브리드**: 기본은 자동 진행, 중요한 두 지점(스토리 확정 · 최종 조립)에서만 사용자 확인.

---

## 1. 목표와 설계 원칙

- **한 흐름, 끊김 없음**: 빈 화면 → 완성된 세로 스크롤 뷰어까지 탭 이동 없이 좌측 파이프라인 레일로 이동.
- **사람은 결정만, 기계는 반복을**: 창작 결정(스토리·캐릭터 룩·최종 검수)은 사람이, 70컷 프롬프트 합성·렌더·조립은 자동.
- **긴 렌더를 견디는 UX**: 35분 렌더 동안 실시간 썸네일·진행률·ETA·부분 재렌더. "제출하고 방치"가 아니라 "지켜보며 손봄".
- **항상 편집 가능**: 자동 생성물(비트·프롬프트·말풍선)은 전부 인라인 편집 가능. 자동은 초안일 뿐.
- **자가완결 산출물**: 최종은 base64 임베드 단일 HTML — 어디서 열어도 안 깨짐(검증된 패턴).

## 2. 정보구조 — 6단계 파이프라인 + 하이브리드 게이트

```
 ┌─ 좌측 파이프라인 레일 (항상 표시, 현재 위치/게이트 상태) ──────────────┐
 │                                                                        │
 │  ①스토리  →  ②스토리보드  →  ③캐스트  →  ④렌더  →  ⑤레터링·조립  →  ⑥익스포트 │
 │   [GATE]        auto          auto/GATE?   auto         [GATE]          │
 └────────────────────────────────────────────────────────────────────────┘

 ● GATE = 사용자 확인 후 진행 (하이브리드 정지점)
   - ① 스토리 확정 게이트  : 비트/대사가 마음에 들 때까지 반복 → "확정" 눌러야 다음
   - ⑤ 최종 조립 게이트    : 말풍선 위치/뷰어 확인 후 "조립·익스포트"
 ● 그 외(②③④)는 자동 진행하되 각 화면에서 언제든 수정 가능
   - ③ 캐스트는 옵션 게이트: 신규 캐릭터 등장 시에만 "룩 확정" 확인을 띄움
```

자동 흐름의 의미: ①에서 "확정"을 누르면 ②비트→샷리스트→프롬프트 합성, ③신규 인물 레퍼런스 등록, ④렌더가 **연쇄로 자동 실행**되고, 사용자는 ④렌더 콘솔에서 결과를 지켜보다 ⑤에서 멈춰 최종 검수한다.

## 3. 화면별 설계

### ① 스토리 스튜디오 ★ (스크립트 작성의 핵심)

AI 보조 집필 + 비트 카드 편집. 산문 대본과 비트시트를 오가며 70컷을 짠다.

```
┌ Webtoon Studio ─ 심야 편의점 / 2화 ───────────────────[ 저장됨 ]─┐
│레일│  ┌─ 입력/지시 ────────┐  ┌─ 비트시트 (편집 가능) ───────────┐ │
│ ①◉│  │ 로그라인           │  │ #1  [wide] [EXT·편의점 야간]     │ │
│ ② │  │ "편의점 단골이      │  │   🅝 비 오는 새벽 3시…            │ │
│ ③ │  │  사실 유령이었다"   │  │ ───────────────────────────────  │ │
│ ④ │  │ 장르 [호러/스릴러]  │  │ #2  [full][CVS] 🧍지호           │ │
│ ⑤ │  │ 톤   [차갑고 적막]  │  │   💭 오늘도… 조용하네.           │ │
│ ⑥ │  │ 분량 [70컷 ▼]       │  │   [+대사] [샷▼] [장소▼] [🗑]     │ │
│   │  │ 화풍 [animagine ▼]  │  │ ───────────────────────────────  │ │
│   │  │ ┌───────────────┐  │  │ #3  [emotion]🧍지호  😱          │ │
│   │  │ │ ✦ AI로 초안 생성│  │  │   🗩 어떻게…                     │ │
│   │  │ └───────────────┘  │  │   …                              │ │
│   │  │ ── 부분 보조 ──    │  │ [＋ 비트 추가]   (70컷 중 12 작성) │ │
│   │  │ • 반전 더 강하게    │  └──────────────────────────────────┘ │
│   │  │ • 이 비트 다시 써   │  ┌─ 미리보기(대본) ──────────────────┐ │
│   │  │ • 침묵 컷 늘려      │  │ 산문 대본 ↔ 비트 동기화 뷰         │ │
│   │  └────────────────────┘  └──────────────────────────────────┘ │
│   │                        [ 임시저장 ]   [ 스토리 확정 → ② ] 🔒GATE│
└──────────────────────────────────────────────────────────────────┘
```

- **AI 집필**: "✦ AI로 초안 생성" → Claude API가 로그라인/장르/톤/분량으로 3막 비트시트 초안. 부분 보조 버튼으로 특정 비트만 재생성("반전 강화", "이 비트 다시").
- **비트 카드**: 각 컷 = `shot / location / character / 장면묘사 / 대사[]`. 드롭다운·인라인 텍스트로 즉시 편집. 대사는 `type(speech/thought/shout/narration/sfx) + speaker + 한글`.
- **양방향 뷰**: 산문 대본 ↔ 비트시트 동기화(작가는 산문으로, 시스템은 비트로).
- **게이트**: "스토리 확정"이 ②~④ 자동 연쇄의 방아쇠.

### ② 스토리보드 / 샷리스트 (자동, 편집 가능)

비트 → 70컷 그리드. 컷마다 자동 합성된 **프롬프트**와 **IP-Adapter 배정**을 보여주고 손볼 수 있다.

```
┌ 스토리보드 (70컷) ───────────────── [모델 일괄변경] [재합성] ─┐
│ #1 wide  EXT      | 프롬프트 ▸ "...neon storefront..."  [no-id]│
│ #2 full  CVS 지호 | 프롬프트 ▸ "...gray hoodie..."  [id:지호/full0.4]│
│ #3 emo   CVS 지호 | 프롬프트 ▸ "...terror..."     [id:지호/face0.5]│
│ #15 ins  CVS      | 프롬프트 ▸ "...still-life...단일객체" [no-id] ⚠소품│
│   ...                                                          │
│ • 샷 바꾸면 IP-Adapter 강도 자동 갱신(wide/full→0.4, close→0.5) │
│ • '말풍선 자리' 힌트로 여백 위치 자동 지정(긍정 프롬프트 bubble 금지)│
│ • ⚠ 소품 인서트는 자동으로 "객체강제" 프롬프트 프리셋 적용        │
└────────────────────────────────────────────────────────────────┘
```

### ④ 렌더 콘솔 ★ (생성 — 긴 렌더를 지켜보는 화면)

70셀 그리드가 렌더되는 대로 실시간 썸네일로 채워진다. WebSocket으로 상태 스트리밍.

```
┌ 렌더 ─ animagine HQ ──────  ▓▓▓▓▓▓▓░░░  43/70  ETA 13:20 ─┐
│ ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐   상태 범례               │
│ │🖼││🖼││🖼││⏳││ ││ ││✕│   🖼완료 ⏳렌더중 □대기 ✕실패│
│ └──┘└──┘└──┘└──┘└──┘└──┘└──┘                              │
│ ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐   • 완료 셀 클릭 → 원본 확대 │
│ │🖼││🖼││🖼││🖼││🖼││🖼││🖼│   • 실패 셀 → [재렌더]      │
│ └──┘└──┘└──┘└──┘└──┘└──┘└──┘   • 마음에 안 듦 → [프롬프트 │
│   ...                            손보고 재렌더]            │
│ [⏸ 일시정지] [실패 5개 일괄 재렌더] [소품컷 객체강제 재렌더] │
└────────────────────────────────────────────────────────────┘
```

- **실시간 채움**: 헬퍼는 폴링 후 일괄 다운로드지만, 콘솔은 `/status`를 구독해 **완료되는 즉시 썸네일** 표시(기존 `.jobids.json` 직접조회 패턴을 백엔드가 수행).
- **부분 재렌더**: 실패/불만 셀만 프롬프트·시드 손봐 재제출. 전체를 다시 돌리지 않는다.
- **품질 토글**: draft로 빠르게 전체 → 확정 후 hq로 최종만.

### ⑤ 레터링 & 조립 ★ (드래그 말풍선 — 최종 게이트)

패널 위에 말풍선을 **드래그로 배치**하고 한글을 인라인 수정. 오버레이는 100% 정확·수정 자유(이미지에 안 굽힘).

## 4. 아키텍처

```
┌ 브라우저 (프론트엔드) ────────────────────────────────────┐
│ 경량 SPA: htmx + Alpine.js (드래그/그리드만 소량 JS)        │
│  · 파이프라인 레일 · 비트 카드 편집 · 렌더 그리드(WebSocket) │
│  · 말풍선 드래그 오버레이 · 세로 스크롤 미리보기            │
└───────────────┬──────────────────────────▲───────────────┘
        REST/JSON│                  WebSocket│(렌더 진행 스트림)
┌───────────────▼──────────────────────────┴───────────────┐
│ FastAPI 백엔드 (로컬)                                      │
│  /api/projects            프로젝트/회차 CRUD (project.json)│
│  /api/script/generate     → Claude API (집필/비트 초안)    │
│  /api/beats               비트 편집·프롬프트 자동합성       │
│  /api/cast/register       → weru_imagegen.register        │
│  /api/render (WS)         → weru_imagegen gen 패턴 래핑     │
│  /api/lettering           lettering.json 편집              │
│  /api/assemble            → build_viewer.py 래핑           │
│  재사용: gen_jobs(비트→jobs), build_viewer, weru_imagegen  │
└───────┬───────────────────────────────┬──────────────────┘
        │ SSH 단접속(submit/status/download)│ HTTPS
┌───────▼────────────────┐     ┌─────────▼─────────┐
│ WeRU.B GPU 서버         │     │ Anthropic Claude  │
│ FastAPI(이미지/IP-Adapter)│   │ API (집필 보조)    │
└─────────────────────────┘     └───────────────────┘
```

- **백엔드는 기존 스크립트의 얇은 래퍼**: 새 렌더 로직을 쓰지 않고 검증된 `weru_imagegen.py`(submit/status/download, 백오프, 키프얼라이브)와 `build_viewer.py`를 모듈로 호출.
- **집필 AI**: 스토리 초안/부분 재생성은 백엔드가 Claude API로 처리(스튜디오는 Claude Code 없이도 독립 구동).
- **상태 저장**: 회차당 `project.json` 하나가 단일 진실원천(메타·캐스트·비트·렌더상태·레터링).

## 5. 데이터 모델 (project.json)

```jsonc
{
  "title": "심야 편의점", "episode": 2,
  "style": { "model": "animagine", "style": "illustration", "quality": "hq" },
  "cast": [ { "name": "지호", "character_id": "e95ac39e…", "ref": "refs/jiho.png" } ],
  "beats": [
    { "id": 1, "act": "A", "shot": "wide", "loc": "EXT", "char": null,
      "scene": "rainy neon storefront…", "bubble_space": "top",
      "dialogue": [ { "type": "narration", "speaker": "", "ko": "비 오는 새벽 3시…" } ] }
  ],
  "render": { "panel_001": { "status": "done", "seed": 2001, "file": "panel_001.png" } },
  "settings": { "negative_preset": "animagine-strong" }

```

`beats`에서 `jobs.json`·`lettering.json`은 **파생 생성**(기존 생성기 로직). 단일 소스 → 충돌 없음.

## 6. 기존 하네스 자산 ↔ 스튜디오 매핑

| 기존 | 스튜디오에서의 역할 |
|------|--------------------|
| `gen_ep01full.py`(비트→jobs+lettering) | `/api/beats` 프롬프트 합성 엔진(BEATS가 DB의 `beats`로 대체) |

| 검증된 운영값(IP-Adapter·강화negative·bubble금지·소품프리셋) | 백엔드 기본값/프리셋으로 코드화 |
| `style-bible.md` 규약 | CSS 말풍선 토큰·품질 프로파일 기본값 |

## 7. 구축 로드맵 (MVP → 풀)

- **M1 — 골격(읽기전용 파이프라인)**: project.json 로드 → ④렌더 콘솔(WebSocket 진행) + ⑥미리보기. 기존 산출물을 웹으로 "보는" 단계.
- **M2 — 생성 루프**: ③캐스트 등록 + ④부분 재렌더 + ⑤말풍선 드래그 편집 + 조립. "손보고 다시 굽는" 핵심 UX.
- **M3 — 집필**: ①스토리 스튜디오(Claude API 집필) + ②자동 합성 + 하이브리드 게이트 연쇄. "빈 화면부터" 완성.
- **M4 — 다회차/운영**: 대시보드, 연속성 승계, 익스포트/배포, 프리셋 관리.

> 권장: **M1→M2 먼저**(이미 검증된 렌더/조립을 GUI로 감싸 즉시 효용) → M3 집필 추가.
